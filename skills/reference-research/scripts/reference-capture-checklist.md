# Reference Capture Checklist

Use this lightweight artifact when running `reference-research`.

## Per-reference capture
1. Record the product or source name.
2. Record where it was found and when it was checked.
3. Write the target user in one sentence.
4. Write the problem it solves in one sentence.
5. Summarize the visible flow in 3-5 bullets.
6. Note the UX pattern or messaging pattern that matters.
7. Record pricing, packaging, or positioning signals if visible.
8. Mark confidence as High / Medium / Low.
9. Mark whether the pattern should be kept, adapted, or avoided.

## Cross-source pass
1. Group repeated patterns across 3 or more references.
2. Separate live-product evidence from concept-only inspiration.
3. Flag any pattern that looks attractive but lacks production evidence.
4. Convert findings into planning decisions, not raw links.

## Output minimum
- 3 or more references
- 1 comparison matrix
- 3 or more cross-source findings
- explicit Keep / Adapt / Avoid recommendation
