# Cache & Archive

## Purpose

Use this reference when the original source is blocked, gone, stale, or only partially visible and an archived or cached copy may still preserve enough evidence for planning.

## Consult When

- The original URL fails repeatedly.
- The page was deleted, changed, or moved.
- The page is paywalled but a historical snapshot may still reveal summary-level context.
- Freshness matters less than preserving prior wording, claims, or structure.

## Trigger Surfaces

- blocked URL
- deleted content
- stale but referenceable article
- historical comparison request
- low-confidence recovery attempt

## What This Reference Adds

- A recovery branch for AMP cache, archive services, and historical snapshots.
- A reminder that cached content is useful mainly for evidence preservation, not for current truth.
- A low-trust fallback path when direct access is unavailable.

## Planning-only Adaptation Rules

- Treat archive/cache methods as routing knowledge, not as plugin dependencies.
- Use archived copies to preserve wording, structure, or historical claims, not as a default primary source.
- Always record that the result came from a cached or archived surface.

## Trust Notes

- Trust is lower than the original page because freshness and completeness are uncertain.
- Prefer the live source whenever it is available.
- Use archive results to support planning judgment, not to overstate certainty.

## Routing Summary

1. Try direct or platform-specific access first.
2. If the source remains unavailable, consider archive/cache recovery.
3. Tag the result as low-trust and historical before using it in planning output.
